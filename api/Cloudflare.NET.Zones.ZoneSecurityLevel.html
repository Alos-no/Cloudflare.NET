Security level for a zone, controlling how aggressive Cloudflare challenges visitors.

Use with SecurityLevel setting. 

[JsonConverter(typeof(JsonStringEnumMemberConverter))]
public enum ZoneSecurityLevel

Fields

[EnumMember(Value = "essentially_off")] EssentiallyOff = 1

Essentially off - only the most grievous offenders are challenged.

Challenges only visitors with the worst reputation. Use when expecting very little bad traffic.

[EnumMember(Value = "high")] High = 4

High security - challenges all visitors showing any suspicious behavior.

Challenges all visitors that have exhibited suspicious behavior within the past 14 days.

[EnumMember(Value = "low")] Low = 2

Low security - challenges only the most threatening visitors.

Challenges visitors that have exhibited threatening behavior within the last 14 days.

[EnumMember(Value = "medium")] Medium = 3

Medium security - challenges both moderate and threatening visitors.

Default setting. Challenges visitors that have exhibited moderately threatening behavior within the last 14 days.

[EnumMember(Value = "off")] Off = 0

Security is completely off - no challenges issued.

Only use this setting if you are implementing your own security measures. Not recommended for production use.

[EnumMember(Value = "under_attack")] UnderAttack = 5

I'm Under Attack mode - challenges all traffic with an interstitial page.

All visitors see a JavaScript challenge page that takes ~5 seconds. Should only be used when your site is experiencing a DDoS attack.
