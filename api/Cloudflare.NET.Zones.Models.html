Namespace Cloudflare.NET.Zones.Models

Classes

ActivationCheckResult

Result of a zone activation check operation.

Returned by TriggerActivationCheckAsync when verifying zone activation. The API only returns the zone ID; use GetZoneDetailsAsync to fetch the updated status.

CreateZoneRequest

Request to create a new Zone in Cloudflare.

Creates a zone for a domain and associates it with an account. The domain must not already be added to Cloudflare in the same or another account.

EditZoneRequest

Request to edit a Zone's properties.

Important: Only one property can be changed per API call. Setting multiple properties in a single request may result in an error.

ListZonesFilters

Filters for listing Zones.

All filter properties are optional. When multiple filters are specified, they are combined according to the Match parameter.

PurgeCacheRequest

Defines the request payload for a cache purge operation. At least one property must be set.

PurgeCacheResult

Represents the result of a successful cache purge operation.

UpdateZoneHoldRequest

Request to update an existing zone hold. Both fields are optional; update only the fields you want to change.

UpdateZoneSettingRequest<T>

Request to update a zone setting.

Zone

Represents the detailed information for a Cloudflare Zone.

A zone represents a domain and its configuration in Cloudflare. This record captures all properties returned by the Cloudflare API for a zone resource.

ZoneAccount

Account information nested within a Zone.

This represents the Cloudflare account that owns the zone.

ZoneAccountReference

Account reference for zone creation.

Specifies which Cloudflare account the zone should be created under.

ZoneHold

Represents the zone hold status and configuration.

A zone hold prevents creation and activation of zones with the same hostname. When include_subdomains is enabled, it also blocks all subdomains and SSL4SaaS Custom Hostnames.

Important: Zone holds are an Enterprise-only feature. The Hold property reflects whether the hold is currently active, which depends on whether HoldAfter is in the past.

ZoneMeta

Metadata for a Zone.

This contains additional configuration and quota information for the zone.

ZoneOwner

Owner information for a Zone.

This represents the user or organization that owns the zone.

ZonePlan

Plan information for a Zone.

This represents the Cloudflare billing plan associated with the zone.

ZonePlanReference

Plan reference for zone editing.

Used to specify a plan when editing zone properties.

ZoneSetting

Represents a zone configuration setting.

The Value property contains the setting's current value, which varies by setting type. Use JsonElement methods to extract the appropriate type (GetString, GetInt32, etc.).

ZoneTenant

Tenant information for a Zone.

This represents the tenant in a multi-tenant Cloudflare configuration.

ZoneTenantUnit

Tenant unit information for a Zone.

This represents the tenant unit in a multi-tenant Cloudflare configuration.

Structs

ZoneSettingId

Represents a zone setting identifier.

Zone settings control various aspects of zone behavior including security, performance, caching, and network configuration.

ZoneStatus

Represents the status of a Cloudflare Zone.

Zone statuses indicate the current state of a zone in Cloudflare's system, such as whether it's active, pending verification, or has been moved/deleted.

ZoneType

Represents the type of a Cloudflare Zone setup.

Zone types determine how DNS and traffic handling are configured:

  • FullDNS is fully hosted by Cloudflare
  • Partial (CNAME)DNS remains with the original provider; only specific records are proxied
  • SecondaryCloudflare acts as a secondary DNS provider
