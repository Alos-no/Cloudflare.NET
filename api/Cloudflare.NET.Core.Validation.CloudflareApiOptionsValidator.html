Validates CloudflareApiOptions to ensure all required configuration is present and valid. This validator is invoked at startup when using ValidateOnStart(), providing immediate feedback for configuration issues rather than waiting for the first API call to fail.

Remarks

This validator checks that the API token is provided, which is required for authentication with the Cloudflare API. Additional validation (such as AccountId for R2 or GraphQL URL for Analytics) is performed by package-specific validators.

The validation errors are designed to be clear and actionable, mentioning "Cloudflare" explicitly so developers can quickly identify the source of configuration issues.

When registered as IValidateOptions<TOptions>, this validator skips named options by default. Factories should use ValidateConfiguration(string?, CloudflareApiOptions, CloudflareValidationRequirements) directly for explicit validation of named client configurations with custom exception types.
